[3조, 팀장 유환희]

## ▶ 이슈사항

- 산출물 정리 방향 재정립
  - 업체 단위의 관리가 용이하도록 기존 단일 관리자 구조를 `최상위 관리자(ROLE_SUPER_ADMIN)`와 `일반 관리자(ROLE_COMPANY_ADMIN)`로 분리하는 방향으로 산출물 전반을 재정리
  - 요구사항정의서, DB설계서, 프로젝트구조도, WBS, 브랜치 전략 문서를 위 권한 구조와 업체(company) 단위 데이터 격리 기준에 맞춰 최신화
  - 번호판 관측 이벤트를 포함한 10개 테이블 기준으로 문서와 DB 방향성을 정합화

- AI 번호판 인식 기준 고도화
  - 기존 YOLOv8 중심 기준을 `YOLO11n + EasyOCR` 기준으로 전환하여 산출물 및 개발 방향을 통일
  - OCR 서버 구조 유지와 함께 fallback 응답 개선
  - 데이터셋 구조, 학습 스크립트, 평가 스크립트를 준비하여 재학습 및 성능 비교 기반 마련
  - 출발/도착 외 고속도로 관측, 휴게소 진입/진출 관측 이벤트 처리 방향까지 포함하도록 확장

- JWT 사용자 인증 및 권한 흐름 반영
  - Spring Security, JWT 토큰 발급/검증 구조 반영
  - 로그인, 회원가입, 승인/거절/정지 흐름을 고려한 사용자 인증 구조 정리
  - `ACTIVE` 상태 관리자만 로그인 가능한 방향으로 권한 및 상태 기준을 적용
  - 프론트엔드에서도 `localStorage`, `Pinia`, `Router Guard` 기반 인증 흐름 연결 준비

- 시뮬레이션 및 운행 제어 로직 구체화
  - 시뮬레이션 시작/중지 로직과 `drive_log` 기반 운행 흐름 반영
  - GPS 수신 API와 연결되는 운행 제어 구조 정리
  - 번호판 관측 이벤트 생성 흐름과 운행 시작/종료 처리 방향 정리
  - 휴식 이벤트 감지, 피로도 점수 계산, 운행 이력 및 대시보드 조회 흐름과 연결 가능한 구조로 확대

- 프론트엔드 1차 작업 진행
  - 공통 레이아웃(`AppLayout`), 상단바(`AppTopbar`), 사이드바(`AppSidebar`) 구조 구현
  - 역할별 메뉴 분기 구조 추가: 최상위 관리자 메뉴 / 일반 관리자 메뉴 구분
  - 로그인 화면, 대시보드 화면, 시뮬레이션 화면, 차량/운전자/이력/통계/임계값 화면에 대한 1차 뷰 구조 생성
  - `axios`, `Pinia`, `router` 기반 API 연동 구조 정리
  - 인증 스토어에서 토큰, 관리자 이름, 역할 정보 저장/삭제 구조 반영
  - 대시보드 화면은 1차 시안 수준에서 KPI, 최근 감지 차량, 경고 알림, 주간 통계 UI 구성
  - 시뮬레이션 화면은 차량/운전자 선택, 시나리오 선택, 시작/종료 버튼, 상태 표시 구조 구현
  - 현재 프론트엔드는 1차 작업 단계이며, 이후 실제 API 연동 및 화면 수정/보완이 추가로 진행될 예정

- 기타 진행된 작업
  - 업체 조회, 차량 CRUD, 운전자 CRUD, GPS 수신, 운행 이력, 대시보드 요약 기능 브랜치 병합 진행
  - Swagger 설정, 공통 예외 처리, Enum 구조 등 백엔드 공통 기반 정리
  - 시뮬레이터는 시나리오 구조와 plate event 흐름을 반영하는 방향으로 정리
