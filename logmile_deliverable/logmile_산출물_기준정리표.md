# logmile 산출물 기준 정리표

- 프로젝트명: `logmile`
- 작성 기준일: `2026.04.30`
- 버전 운영 원칙: `초안 및 정리 단계는 v1.x 유지`, `초안 완료 후 v2.x로 전환`
- 관리 원칙:
  - `md`: Git 기준 원본 문서
  - `docx`: 보고/제출용 문서
  - `md`와 `docx`는 같은 기준 내용으로 최신화한다.

## 1. 기준 원칙

| 구분 | 기준 |
|---|---|
| 버전 정책 | 현재 산출물은 `v1.x` 기준으로 유지한다. |
| 문서 원본 | 내용 수정의 기준 원본은 `md`로 관리한다. |
| 제출 문서 | 보고 및 제출 시 `docx`를 사용한다. |
| 동기화 원칙 | 기준 문서(`md`) 수정 시 대응 `docx`도 같은 내용으로 반영한다. |
| 문서 상태 표기 | `초안` 표기가 있는 문서는 아직 확정 전 문서로 본다. |

## 4. 현재 기준 판정

| 항목 | 현재 기준 |
|---|---|
| 문서 체계 | `md` 중심 관리 + `docx` 제출 동기화 |
| 버전 체계 | `v1.x` 유지 |
| 핵심 기준선 | `프로젝트개요서`, `요구사항정의서`, `DB설계서`, `프로젝트구조도` |
| 초안 문서 처리 | 초안 완료 전까지 `v1.x` 안에서 보완 |
| 구버전 문서 처리 | `v1.4` 및 판본 불명 docx는 기준 여부 재판정 필요 |

## 5. 바로 이어서 할 작업

1. 핵심 기준 문서 4종의 제목/버전/기준일을 `v1.x` 체계로 통일한다.
2. `md` 4종과 대응 `docx`의 목차 및 용어를 맞춘다.
3. `docx/logmile_ERD.docx`, `docx/logmile_테이블정의서.docx`의 기준 판본 여부를 확인한다.
4. `v1.4` docx를 보관용으로 둘지 정리 대상으로 둘지 결정한다.
5. 임시 파일과 시스템 파일(`~$`, `.DS_Store`) 정리 여부를 결정한다.
