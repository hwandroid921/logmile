## 4. 우선순위 메모

- **최우선**: `회원가입(BE-004a) → 승인(BE-004b) → 로그인(BE-003) → TenantAccess` 흐름 확정 후 차량/운전자/GPS 개발 진행
- GPS 기반 피로도 판단이 프로젝트 핵심이므로 `GPS 수신 API → 시뮬레이터 전송 → 휴식 판단 → 피로도 계산 → 대시보드 표시` 흐름을 최우선 통합 대상으로 둔다.
- 번호판 인식은 출발/도착 검증, 고속도로 관측, 휴게소 진입/진출 기반 휴식 보조 검증까지 포함하는 운행 신뢰도 검증 기능으로 통합한다.
- 미등록 차량 감지, 동일 번호판 중복 운행 감지, 경로 이탈 감지, 번호판 위변조 의심, OCR 통계는 향후 확장 기능으로 검토한다.
- 제출 전에는 A 정상, B 주의, C 위험 시나리오를 각각 시연 가능한 seed/시뮬레이션 데이터로 준비한다.
- 최상위 관리자 seed 계정 및 샘플 업체/일반 관리자 seed 데이터를 `seed.sql`에 포함한다.
