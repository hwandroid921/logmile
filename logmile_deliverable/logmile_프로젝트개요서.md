# 프로젝트 개요서

## 운행 데이터 기반 화물차 운전자 피로도 실시간 모니터링 플랫폼

- 프로젝트명: `logmile`
- 버전: v4.0
- 기준 산출물: `logmile_프로젝트개요서_v4.docx`, `AGENTS.md`
- 작성 기준일: 2026.04.28
- 개발 기간: 2026.04.28 ~ 2026.05.27 제출 / 2026.06.02 발표
- 팀 구성: 백경서, 유환희
- 프로젝트 성격: 학습 및 연구용 웹 기반 모빌리티 관제 서비스
- 관제 대상: 가상 물류센터 10개사 × 화물차 5대 (총 50대)

## 1. 프로젝트 목적

`logmile`은 번호판 인식, GPS 시뮬레이션, 운행 데이터 분석을 결합하여 화물차 운전자의 피로 위험을 실시간으로 관제하는 웹 기반 모빌리티 플랫폼이다.

기존의 단순 급감속 중심 위험 판단에서 벗어나 연속 운행 시간, 휴식 여부 및 휴식 시간, 일일 총 운행 시간, 야간 운행 시간을 통합 점수화하여 운전자 상태를 정상, 주의, 위험 등급으로 산정한다.

본 프로젝트는 실제 운영 배포가 아닌 학습 및 연구 목적의 시연 가능한 시스템 구축을 목표로 하며, 실제 GPS 단말기와 외부 지도 API 없이 자체 GPS 시뮬레이터 데이터를 기반으로 동작한다.

## 2. 핵심 목표

- YOLO11n + EasyOCR 기반 화물차 번호판 인식 기능 구현
- 번호판 인식 신뢰도가 낮을 경우 수동 입력 fallback 제공
- Python GPS 시뮬레이터를 통한 시나리오 A/B/C 운행 데이터 생성
- Spring Boot 백엔드에서 피로도 점수 계산 및 피로 이벤트 저장
- Vue.js 대시보드에서 차량별 피로 점수, 등급, 판단 근거 실시간 표시
- 관리자 페이지에서 차량, 운전자, 피로도 임계값 관리
- PostgreSQL 기반 운행, GPS, 휴식, 피로 이벤트 데이터 저장

## 3. 시스템 구성

| 계층 | 기술 | 주요 역할 |
|---|---|---|
| AI 서버 | FastAPI, YOLO11n, EasyOCR | 출발/도착 차량 번호판 인식 |
| GPS 시뮬레이터 | Python 3.11 | 운행, 휴식, 야간 운행 패턴이 포함된 GPS 데이터 생성 |
| 백엔드 | Java 21, Spring Boot 3.5, Spring Data JPA, Spring Security + JWT | REST API, 인증/인가, 피로도 계산, 데이터 저장 |
| 프론트엔드 | Vue.js 3, Vite, Pinia, Axios, Chart.js | 관제 대시보드, 관리 화면, 통계 시각화 |
| 데이터베이스 | PostgreSQL 16 | 운행, GPS, 휴식, 피로 이벤트 저장 |
| 인프라 | Docker, Docker Compose | 서비스 실행 환경 구성 |

## 5. 주요 기능

### 5.1 번호판 인식

- 출발 차량 번호판 이미지 업로드 또는 촬영 데이터 입력
- YOLO11n으로 번호판 영역 탐지
- EasyOCR로 번호판 문자 추출
- 인식 신뢰도 0.85 미만 시 수동 입력 fallback 처리
- 도착 시 번호판 재인식을 통한 운행 종료 처리

### 5.2 GPS 시뮬레이션

- 시나리오 A: 정상 운행
- 시나리오 B: 주의 운행
- 시나리오 C: 위험 운행
- 운행 시간, 속도, 휴식 패턴, 야간 운행 패턴이 포함된 GPS 데이터 생성
- 생성된 GPS 데이터를 백엔드 API로 HTTP POST 전송

### 5.3 피로도 판단

- 연속 운행 시간 계산
- 유효 휴식 판단
- 일일 총 운행 시간 누적
- 야간 운행 시간 누적
- 휴식 부족 및 휴식 보정 반영
- 피로도 점수와 정상/주의/위험 등급 산정
- 피로도 판단 근거를 `fatigue_event.reason`에 저장

### 5.4 관제 대시보드

- 운행 중 차량 수, 주의/위험 차량 수, 완료 운행, 평균 피로 점수 표시
- 차량별 현재 속도, 운행 시간, 야간 운행 시간, 휴식 횟수, 피로 점수 표시
- 정상/주의/위험 등급 배지 표시
- 차량 상세 패널에서 판단 근거와 이벤트 타임라인 확인
- 위험 차량 운전자 전화 연결 링크 제공
- 5초 간격 polling으로 차량 상태 갱신

### 5.5 관리 및 통계

- 차량 등록, 조회, 수정, 삭제
- 운전자 등록, 조회, 수정, 삭제 및 차량 배정
- 피로도 임계값 설정
- 운행 이력 목록 및 상세 조회
- Chart.js 기반 일별 운행 시간, 야간 운행 시간, 휴식 누락, 평균 피로 점수 통계 제공

## 6. 피로도 판단 모델

```text
피로도 점수 =
  연속 운행 시간 점수
+ 일일 총 운행 시간 점수
+ 야간 운행 시간 점수
+ 휴식 부족 점수
- 휴식 보정 점수
```

| 항목 | 기준 | 점수 |
|---|---|---:|
| 연속 운행 시간 | 90분 이상 | +10 |
| 연속 운행 시간 | 120분 이상 | +25 |
| 연속 운행 시간 | 180분 이상 | +45 |
| 연속 운행 시간 | 240분 이상 | +65 |
| 일일 총 운행 시간 | 6시간 이상 | +15 |
| 일일 총 운행 시간 | 8시간 이상 | +30 |
| 일일 총 운행 시간 | 10시간 이상 | +45 |
| 야간 운행 시간 | 22:00~06:00 누적 30분 이상 | +10 |
| 야간 운행 시간 | 22:00~06:00 누적 1시간 이상 | +20 |
| 야간 운행 시간 | 22:00~06:00 누적 2시간 이상 | +35 |
| 휴식 부족 | 2시간 운행 후 15분 미만 휴식 | +10 |
| 휴식 부족 | 휴식 누락 2회 이상 | +25 |
| 휴식 보정 | 15분 이상 유효 휴식 | -10 |
| 휴식 보정 | 30분 이상 충분 휴식 | -20 |

| 등급 | 점수 범위 | 조치 |
|---|---:|---|
| 정상 | 0~39 | 모니터링 |
| 주의 | 40~69 | 주의 배지 표시 |
| 위험 | 70 이상 | 위험 배지 표시 및 전화 권고 |

## 7. 주요 DB 설계 방향

| 테이블 | 주요 역할 |
|---|---|
| `admin` | 관리자 계정 및 권한 정보 저장 |
| `driver` | 운전자 기본 정보 및 차량 배정 정보 저장 |
| `vehicle` | 차량 번호, 차종, 활성 상태 저장 |
| `drive_log` | 운행 시작/종료, 시나리오, 운행 상태 저장 |
| `gps_data` | GPS 좌표, 속도, 기록 시각 저장 |
| `rest_event` | 휴식 시작/종료 시각, 휴식 시간, 휴식 유형 저장 |
| `fatigue_event` | 피로 점수, 등급, 판단 항목, 판단 근거 저장 |
| `fatigue_threshold` | 피로도 점수 모델 임계값 key/value 관리 |

## 10. 결론

`logmile` v4는 번호판 인식과 GPS 시뮬레이션을 기반으로 운행, 휴식, 야간 운행 데이터를 통합 분석하는 화물차 운전자 피로도 관제 시스템이다.

본 프로젝트는 실시간성, 설명 가능성, 시연 가능성을 핵심 가치로 하며, 학습 및 연구 목적에 맞춰 자체 생성 데이터를 기반으로 전체 관제 흐름을 구현한다.
