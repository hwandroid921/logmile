| 구분 | 기술 |

| 구성 요소 | 기술 | 역할 |

## 4. 관리자 권한 구조

| 권한 | Role 값 | 주요 기능 |
|---|---|---|
| 최상위 관리자 | `ROLE_SUPER_ADMIN` | 일반 관리자 승인/거절/정지/해제, 전체 업체 목록 조회 |
| 일반 관리자 | `ROLE_COMPANY_ADMIN` | 소속 업체 차량/운전자/운행 관제, 피로도 임계값 설정 |

| 상태 | 설명 |
|---|---|
| `PENDING` | 일반 관리자 회원가입 후 최초 상태. 로그인 불가 |
| `ACTIVE` | 최상위 관리자 승인 완료. 정상 로그인 및 서비스 이용 가능 |
| `REJECTED` | 최상위 관리자 거절 처리. 재가입 불가 |
| `SUSPENDED` | 최상위 관리자 정지 처리. 로그인 차단. 해제 시 `ACTIVE` 복귀 |

## 8. 브랜치 전략

브랜치 구조는 `main → dev → feature/{layer}-{기능명}` 기준으로 운영한다.

| 브랜치 | 역할 |
|---|---|
| `main` | 최종 릴리즈 전용 (직접 커밋 금지) |
| `dev` | 팀 통합 개발 브랜치 (`feature/*` 머지 대상) |
| `feature/ai-*` | AI 서버 및 모델 학습/평가 기능 브랜치 |
| `feature/be-*` | Backend 기능 브랜치 |
| `feature/fe-*` | Frontend 기능 브랜치 |
| `feature/sim-*` | GPS Simulator 기능 브랜치 |
| `feature/infra-*` | Infra 공통 브랜치 |
| `release/*` | 제출 전 QA 브랜치 |

## 10. 환경변수 및 보안 규칙

- `.env` 파일은 절대 Git에 커밋하지 않는다.
- 모든 모듈에 `.env.example` 파일을 함께 제공한다.
- 민감 정보(DB 패스워드, JWT 시크릿, API 토큰)는 반드시 환경변수로 관리한다.
- Spring Boot는 `application-local.yml`로 분리 관리하고 Git에는 포함하지 않는다.
- `.vscode/`, `.claude/`, `__pycache__/`, `node_modules/`, `dist/`, YOLO 모델 가중치(`*.pt`, `*.pth`)는 Git에 커밋하지 않는다.

## 11. 핵심 우선순위 메모

- 최우선 흐름은 `회원가입 → 승인 → 로그인 → TenantAccess 확정` 이후 차량/운전자/GPS 기능으로 이어진다.
- GPS 기반 피로도 판단이 프로젝트 핵심이므로 `GPS 수신 API → 시뮬레이터 전송 → 휴식 판단 → 피로도 계산 → 대시보드 표시` 흐름을 우선 통합한다.
- 번호판 인식은 운행 시작/종료 자동화와 관측 이벤트 정합성을 위한 기능으로, GPS/피로도 흐름 안정화 후 연동한다.
- 제출 전 A(정상), B(주의), C(위험) 시나리오별 시연 데이터와 seed 데이터를 준비한다.
